# v2.2 Transaction Context Migration

v2.2 adds `CapOperationContext` as the foundation for transaction-aware publish
integration across storage adapters.

Existing `tx` style remains supported:

```ts
await cap.publish('user.created', payload, { tx: em });
```

New code can pass the operation context:

```ts
await cap.publish('user.created', payload, { ctx: { tx: em } });
```

If both `ctx` and `tx` are provided, `ctx` is used. This lets integrations attach
transaction data, post-commit hooks, rollback hooks, or metadata without adding
more top-level publish options.

When `tx` or `ctx.tx` is provided, CAP saves the outbox row inside that
transaction and defers broker emit by default. The scheduler dispatches after
commit. Use `immediate: true` only when intentionally attempting broker emit in
the same call.

Storage adapters should accept the new `savePublish(event, ctx?)` signature.
Adapters that still implement `savePublishWithTx(event, tx)` continue to work as
deprecated compatibility.
